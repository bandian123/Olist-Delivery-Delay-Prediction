# Olist 订单物流延误预测
## 项目概述
本项目旨在解决电商平台订单物流延误问题，通过分析巴西 Olist 平台的订单数据，构建机器学习模型预测订单是否会延误送达。项目从数据清洗、数据探索性分析（EDA），同时利用matplotlib和tableau进行了数据可视化，特征工程到模型构建与优化，完成了一套完整的预测分析流程，最终模型在测试集上达到 0.7397 的 AUC。
## 数据来源
数据来自Kaggle的Olist提供的巴西电子商务公共数据集，包含2016-2018年间约10万笔订单的真实数据。原始数据分散在9张表中，包括订单信息、产品信息、客户信息、卖家信息、支付信息、评价信息等。数据链接：https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce
## 分析流程
### 1. 数据探索与预处理
数据合并：将 9 张表通过主键关联，构建统一的分析数据集
缺失值处理：订单处理时间、体积、重量缺失值很少，直接删去对应的缺失列,距离列使用划分后的训练集中相同的商家地区和客户地区列的距离的平均值填充
异常值处理：极端值（如超远距离、超高运费）与延误率存在相关性，选择保留
### 2.特征工程
创建体积、重量、处理时长等数值特征
提取订单时间特征（小时、星期、月份）
通过用户与商家的经纬度大致计算了订单的运输距离
对商品类别和州代码进行了高层次聚合，映射了11个类别以及5个地区
计算订单总金额和总运费
### 3.可视化分析
分析了延误率与各特征的关系：
距离、运费、体积、重量与延误率呈正相关
处理时长与延误率正相关
二月、三月（巴西狂欢节）、十一月（黑五）延误率较高
凌晨 3-5 点下单延误率最低（订单量少，处理效率高）
北部/东北部地区（亚马逊雨林）延误率最高
美妆健康类产品延误率最高，宠物用品最低
### 4.模型构建
特征选择：12 个特征（产品类别、体积、重量、处理时间、距离、运费总额、订单总额、购买小时、星期、月份、客户地区、卖家地区），后又尝试选择订单对应的商家的平均延误率作为特征，发现模型过拟合，故删去该特征。
数据划分：75% 训练集，25% 测试集，分层采样保持类别平衡
基础模型：LightGBM（初始 AUC 0.7289）
Optuna 超参数调优：100 次试验，5 折交叉验证，最佳 AUC 0.7352
对比实验：XGBoost 调优后测试集 AUC 0.7397，与 LightGBM 接近，选择效率更高的 LightGBM
### 5.特征重要性分析
基于信息增益的特征重要性排序：
购买月份（33.2%）
运输距离（14.0%）
运费总额（9.5%）
处理时间（8.1%）
体积（6.6%）
单特征 AUC 均低于 0.57，说明模型成功捕捉了特征间的交互效应。
## 主要结论
核心影响因素：月份、距离、运费是延误的主要预测因子，狂欢节和黑五等促销节点物流压力显著
地域差异：北部/东北部地区因地理条件（亚马逊雨林）延误率较高
时效特征：处理时长和运输距离与延误率正相关，凌晨下单效率更高
模型表现：LightGBM 在测试集上达到 AUC 0.738，模型水平良好，未达到0.8的原因主要是因为并没有强相关特征，无法得到对应的物流公司，线路，以及当时天气情况。
